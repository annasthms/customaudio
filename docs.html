customaudio.js

A plugin that allows for more customization of audio posts.

Built: August 9, 2020

Author: annasthms

Requirements: a tumblr blog

Installation

Either before </head> or before </body> (basically wherever you add your scripts), paste

<script src="https://cdn.jsdelivr.net/gh/annasthms/customaudio@latest/customaudio.min.js"></script>

The default settings use feather icons and so, if you'd like to simply use the default settings, you must include the feather icons before the customaudio script: 

<script src="https://unpkg.com/feather-icons"></script>

Basic Usage

Instantiate and call the function either inside a $(document).ready(); or after all the code for the audio posts:

customAudio();

Examples:

<script src="https://cdn.jsdelivr.net/gh/annasthms/customaudio@latest/customaudio.min.js"></script>
<script>
$(document).ready(function() {
	customAudio();
});
</script>
[post html]

<script src="https://cdn.jsdelivr.net/gh/annasthms/customaudio@latest/customaudio.min.js"></script>
<script>customAudio();</script>

Advanced Usage

For those who want more control over the audio and customization—for example, those who would like audio to be customized with different options depending on the post. A general knowledge of JavaScript classes (or object-oriented programming) is recommended.

The customAudio function utilizes a class named CustomAudio to keep track of and localize all the relevant data and methods. CustomAudio has the following methods:

Signature Description Parameters
constructor(options) Initializes the CustomAudio class and overwrites the default settings. This is called when new CustomAudio() is called. options an object containing the settings (see options for details)
update(options) Updates the settings with the new settings. options an object containing the new settings (see options for details)
pause_all() Pauses all customize audio created with this instance of the class. none
customize_iframe(iframe, post) Customizes the audio associated with the iframe.

iframe the audio iframe to customize (must be from {AudioPlayer}

post the post containing the iframe

* both must be DOM elements, not strings/etc.
create() Customizes all audio iframes that can be found through the settings given when instantiating (or updating). none

Here is an outline of how these methods may be used:

// set the settings
const settings = {
	... // some settings
};
// instantiate the CustomAudio object
const ca = new CustomAudio(settings);

// do some stuff

ca.create(); // customize the audios

// do some more stuff

ca.update({
	// some updated settings (e.g. update the post selector or array)
});

ca.create(); // customize the new audios

Function Option Defaults

customAudio({
	post: ".post",
	wrappers: {
		audio: ".custom_audio_wrapper",
		buttons: ".custom_audio_buttons",
		duration: ".custom_audio_duration",
		timeCurrent: ".custom_audio_current_time",
		timeLeft: ".custom_audio_time_left",
		seekbar: ".custom_audio_seekbar"
	},
	default: false,
	pauseAll: true,
	playButton: "<i data-feather='play'></i>",
	pauseButton: "<i data-feather='pause'></i>",
	errorIcon: "<i data-feather='x'></i>",
	hideInfoIfError: true,
	callAfterLoad: () => {
		feather.replace();
	}
});

Function Options

Name Description Options
post The posts' CSS selector. Alternatively, an array containing the DOM elements of the posts.

"selector" or an array of DOM elements

Example: "article.post", ".entry", $(".post").toArray()
wrapper The CSS selectors for where to fill in with the relevant audio information. Replaces the {audio:___}s from the first iteration of customAudio.js. an object (see below)
audio The CSS selector for the wrapper for the new <audio> element. If no element with this selector is found, the <audio> will be inserted right after the original iframe.

"selector"

Example: ".custom_audio_wrapper"
buttons The CSS selector for the wrapper for the play/pause buttons or error icon. Replaces {audio:AudioButtons}.

"selector"

Example: ".custom_audio_buttons"
duration The CSS selector for the wrapper for the total duration of the audio. Replaces {audio:PlayTimeLeft}.

"selector"

Example: ".custom_audio_duration"
timeCurrent The CSS selector for the wrapper for the current time of the audio (how much of the audio has been played). Replaces {audio:CurrentPlayTime}.

"selector"

Example: ".custom_audio_current_time"
timeLeft The CSS selector for the wrapper for how much time is left of the audio. Replaces {audio:PlayTimeLeft}.

"selector"

Example: ".custom_audio_time_left"
seekbar The CSS selector for the wrapper of the seekbar. Replaces {audio:Seekbar} (to an extent—see seekbar for detailes).

"selector"

Example: ".custom_audio_seekbar"
default

If true, the audio player will simply be the default audio player provided by the brower.

If false, the {audio:___}s will be customizable.

true or false
pauseAll

If true, all other audio posts will be paused when one plays.

If false, any audio post that is currently playing will continue to play, even if a different one has been clicked to play.

true or false
playButton

The HTML for the play button icon. (The default value assumes that you have feather icons also included.)

HTML (in quotes)

Example: "<i data-feather='play'></i>", "<i class='th th-play'></i>"
pauseButton

The HTML for the pause button icon. (The default value assumes that you have feather icons also included.)

HTML (in quotes)

Example: "<i data-feather='pause'></i>", "<i class='th th-pause'></i>"
errorIcon

The HTML for the error icon (for when the audio cannot be loaded). (The default value assumes that you have feather icons also included.)

HTML (in quotes)

Example: "<i data-feather='x'></i>", "<i class='th th-cross'></i>"
hideInfoIfError

If true, all information related to time (time left, current time, and total time) is hidden.

If false, all information related to time (time left, current time, and total time) is filled in with N/A. All the information is in a <span>: time left is in span.anna_audio_time_left, curren time is in span.anna_audio_current_time, and total time is in span.anna_audio_duration.

true or false
callAfterLoad

The function that gets called after the audio has been loaded.

Note: If you don't want any function to be called, set this to null.

A function

Re: The Seekbar

customAudio adds a "progress bar" within the seekbar wrapper whose width is equal to the percentage of the duration of the audio that has been played. This progress bar has the class custom_audio_seekbar_progress.

You must style the seekbar. The parts are there, but you can not see it without styling the seekbar as it has no height, width, background color, etc.

Here is some very basic seekbar styling: 

/* .custom_audio_seekbar is the wrapper for the seekbar set when calling customAudio */
.custom_audio_seekbar {
	width      : 200px;
	height     : 5px;
	background : #dfdfdf;
}

.custom_audio_seekbar_progress {
	width      : 0;
	height     : 100%;
	background : #07362c;
}

A Very Basic Example

HTML

{block:Audio}
	{block:AudioPlayer}
	{AudioPlayer}
	<div class="audio_player">
		<span class="custom_audio_buttons"></span>
		<span class="custom_audio_current_time"></span>
		<div class="custom_audio_seekbar"></div>
		<span class="custom_audio_time_left"></span>
	</div>
	<div class="audio_info">
		{block:Artist}<p>artist: {Artist}</p>{/block:Artist}
		{block:Album}<p>album: {Album}</p>{/block:Album}
		{block:TrackName}<p>track name: {TrackName}</p>{/block:TrackName}
		<p>length: <span class="custom_audio_duration"></span></p>
	</div>
	{/block:AudioPlayer}
{/block:Audio}

.audio_player > div {

}

	height     : 0.5em;
	background : #dfdfdf;
}

	width      : 0;
	height     : 100%;
	background : #07362c;
}

	content : "-";
}

.audio_info p {
	margin : 0.25em 0;
